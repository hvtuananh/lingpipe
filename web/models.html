What are Models?

For each language, genre, and training corpus, LingPipe is able to generate a range of models based on task (e.g. named-entity recognition or part-of-speech tagging) and training parameters. These models are then loaded at run time by LingPipe where they provide language-, genre- and task-specific details to the core LingPipe API.

Model Naming Convention

The models are all serialized objects. A name such as ne-en-news-muc6.TokenShapeChunker indicates that the task is named-entity recognition (ner), the language is English (en), the genre of text it applies to is news (news), and the corpora on which it was trained is the MUC-6 corpus (muc6). The file suffix is the (unqualified) name of the class of object represented in the model.
